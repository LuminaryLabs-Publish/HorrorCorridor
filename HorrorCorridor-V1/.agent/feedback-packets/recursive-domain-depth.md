# Natural Recursive Domain Depth

Status: active

## User Direction

- The earlier tree was too mechanical. Algorithms, implementation steps, adapters, rendering concerns, and proofs must not be relabeled as domains.
- A domain is a soft-bounded natural subject that owns coherent state, meaning, reset, snapshot, inputs, outputs, and relationships.
- Deepening asks which independently meaningful natural child boundaries exist inside the parent, then repeats the same test for each accepted child.
- A deep path is valid only while every level owns a narrower truth. Depth has no hard cap, but it is never a target.
- A node may have one child or several siblings when those boundaries are genuinely distinct.
- A domain may be a leaf. Do not force every domain path to terminate in an atomic kit.
- Behaviors belong to kits, stable operations belong to services, platform glue belongs to hosts, and validation belongs to proofs.

## Application

Use `.agent/domain-depth-examples.md` as the corrected reference set and `.agent/horror-corridor-domain-tree.md` as the canonical composition. Future sprint planning should select one natural domain boundary, then choose its kits and services without converting their internals into more domains.
